## SCORING METHODOLOGY

**Defensibility Score (1-10):**
- **1-2:** Generic/commodity вҖ” dozens of competitors, no moat
- **3-4:** Minor differentiation вҖ” some unique features but easily replicated
- **5-6:** Moderate defensibility вҖ” meaningful proprietary integrations, some data moats
- **7-8:** Strong defensibility вҖ” proprietary engine dependencies, unique data flows, significant switching costs
- **9-10:** Category-defining вҖ” no prior art, creates new market category, deep ecosystem dependency

**Factors Evaluated:**
1. Prior art density (fewer competitors = higher score)
2. Proprietary engine connections (more = higher score)
3. Unique data flows (irreplicable data = higher score)
4. Ecosystem dependency (harder to extract = higher score)
5. Novel features count (more unique features = higher score)
